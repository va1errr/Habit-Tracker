# US-003 - Habit Completions Data Model

## Table: habit_completions

| Column            | Type     | Constraints                                   | Purpose                                    |
|-------------------|----------|-----------------------------------------------|--------------------------------------------|
| `id`              | `BIGINT` | PRIMARY KEY, GENERATED BY DEFAULT AS IDENTITY | однозначное определение факта выполнения   |
| `habit_id`        | `BIGINT` | NOT NULL, FOREIGN KEY REFERENCES habits(id)   | однозначное определение связанной привычки |
| `completion_date` | `DATE`   | NOT NULL                                      | дата выполнения привычки                   |

## Database Constraints

- уникальное ограничение `uq_habit_completions_habit_date` на `(habit_id, completion_date)` запрещает повторное
  выполнение привычки за одну дату

## Application Rules

- текущая дата вычисляется приложением в часовом поясе `Europe/Moscow`
- архивирование привычки не удаляет сохранённые факты выполнения
