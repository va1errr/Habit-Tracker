# US-001 - Создание привычки

## User Story
**Как** пользователь,  
**я хочу** создать ежедневную привычку,  
**чтобы** отслеживать её выполнение.

## Acceptance Criteria

### Сценарий 1 — Создание привычки с названием и описанием

**Given** привычки с таким названием не существует  
**When** пользователь создаёт привычку с названием и описанием  
**Then** привычка создаётся  
**And** привычка имеет статус «активна»  
**And** сохраняются переданные название и описание

### Сценарий 2 — Создание привычки без описания

**Given** привычки с таким названием не существует  
**When** пользователь создаёт привычку только с названием  
**Then** привычка создаётся  
**And** привычка не имеет описания

### Сценарий 3 — Название не указано

**Given** пользователь не указал название  
**When** пользователь пытается создать привычку  
**Then** привычка не создаётся  
**And** пользователь получает сообщение об обязательном названии

### Сценарий 4 — Название состоит из пробельных символов

**Given** пользователь указал название, состоящее только из пробельных символов  
**When** пользователь пытается создать привычку  
**Then** привычка не создаётся  
**And** пользователь получает сообщение об обязательном названии

### Сценарий 5 — Пробелы по краям названия

**Given** привычки с таким названием не существует  
**When** пользователь создаёт привычку с пробельными символами в начале или конце названия  
**Then** привычка создаётся  
**And** пробельные символы в начале и конце сохранённого названия отсутствуют

### Сценарий 6 — Повторяющееся название

**Given** существует активная или архивированная привычка с таким названием без учёта регистра  
**When** пользователь пытается создать привычку с тем же названием  
**Then** привычка не создаётся  
**And** пользователь получает сообщение о повторяющемся названии
